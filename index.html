Modularity

FP Glue

"[T]o increase one's ability to modularize a problem conceptually, one must provide new kinds of glue in the programming language. ... [F]unctional languages provide two new, very important kinds of glue." - John Hughes

  • Lazy evaluation
  • Higher-order functions

Higher-Order Functions (HOFs)

  • Take one or more functions as arguments, and/or
  • Return a function as a result

Building Functions

  • Function composition: h(x) = f(g(x))
  • Partial application: f(1,2) //g(x,y,z)
  • Currying: f(1) //g(x)

Map

Modus Operandi

  • Applies given function to every element in a data structure
  • Produces a new data structure
  • Doesn't change the shape of the data structure

Imperative vs Functional

var list = ['Map', 'Filter', 'Fold'];

var imperative = function(list) {
    result = [];
    for(var i = 0; i < list.length; i++) {
        result.push(list[i].charAt(0));
    }
    return result;
}

var functional = function(list, func) {
    return _.map(list, func);
}

imperative(list);
// ['M', 'F', 'F']
functional(list, function(str) { return str.charAt(0) });
// ['M', 'F', 'F']

Language Support

Language Native Support Syntax Notes
Clojure Yes (map func list)
JavaScript 1.6 array.map(func) Function is passed element, its index and array.
Python Yes map(func, list) Returns list in 2, iterator in 3.
Ruby Yes enum.map {block}

Defining Map


var cons = function(element, list) {
    return [element].concat(list);
}

var map = function(func, list) {
    if (list.length === 0) return [];
    return cons(func(list[0]), map(func, list.slice(1)));
}

* Defined recursively for illustrative purposes. See Recursion chapter in Functional JavaScript for information on limitations of recursion in JS, and workarounds.

Filter

Modus Operandi

  • Applies a predicate to every element in a data structure
  • Produces a new data structure containing only those elements for which the predicate function returns true

Imperative vs Functional

var list = ['Map', 'Filter', 'Fold'];
var imperative = function(list, letter) {
    result = [];
    for(var i = 0; i < list.length; i++) {
        if (list[i].charAt(0)
                   .toLowerCase() === letter.toLowerCase()) {
            result.push(list[i]) 
        }
    }
    return result;
}
var functional = function(list, pred) {
    return _.filter(list, pred);
}
var startsWith = function(l) { 
    return function(s) {
        return s.charAt(0).toLowerCase() === l.toLowerCase();
    }
}
imperative(list, 'f');             // ['Filter', 'Fold']
functional(list, startsWith('f')); // ['Filter', 'Fold']

Language Support

Language Native Support Syntax Notes
Clojure Yes (filter predicate list) Can also filter with list comprehensions.
JavaScript 1.6 array.filter(pred)
Python Yes filter(func, list) Returns list or iterator (3.x). Also has list comprehensions.
Ruby Yes enum.map {block}

Source: Wikipedia

Defining Filter


var cons = function(element, list) {
    return [element].concat(list);
}

var filter = function(pred, list) {
    if (list.length === 0) return [];
    return pred(list[0]) 
      ? cons(list[0], filter(pred, list.slice(1)))
      : filter(pred, list.slice(1));
}

* Defined recursively for illustrative purposes. See Recursion chapter in Functional JavaScript for information on limitations of recursion in JS, and workarounds.

Fold/Reduce

Modus Operandi

  • A family of functions
  • AKA accumulate, aggregate, compress or inject
  • Processes a recursive data structure and reduces it to a single value, which could itself be a data structure
  • Applies a combining function to each element of the structure and a return value, or accumulator, that is built up as the structure is processed
  • Accumulator can be given a starting value
  • Can be left associative or right associative
  • Can be strict or lazy

Recursive Pattern


var sum = function(list, initVal) {
    if (list.length === 0) { return initVal }
    return sum(list.slice(1), list[0] + initVal); 
}
sum([1, 2, 3], 0); // 6

var length = function(list, initVal) {
    if (list.length === 0) { return initVal }
    return length(list.slice(1), 1 + initVal); 
}
length([1, 2, 3], 0); // 3

var reverse = function(list, initVal) {
    if (list.length === 0) { return initVal }
    return reverse(list.slice(1), [list[0]].concat(initVal));
}
reverse([1, 2, 3], []); // [3, 2, 1]

Defining Fold


var foldL = function(func, initVal, list) {
    if (list.length === 0) { return initVal }
    return foldL(func, func(initVal, list[0]), list.slice(1))
}
foldL(function(acc, x) { return acc + x }, 0, [1,2,3]);
// 6

var foldR = function(func, initVal, list) {
    if (list.length === 0) { return initVal }
    return func(list[0], foldR(func, initVal, list.slice(1)))
}
foldR(function(x, acc) { return x + acc }, 0, [1,2,3]);
// 6

Fold Left vs Fold Right

Fold Left Fold Right
Left associative Right associative
Never works with infinite structures Can potentially work with infinite structures
Can be implemented with Fold Left for finite structures

Defining Map/Filter with Fold

var list = ['Map', 'Filter', 'Fold'];
var cons = function(element, list) {
    return [element].concat(list);
}

var map = function(func, list) {
    return foldR(function(x, acc) { 
        return cons(func(x), acc) 
    }, [], list)
}
map(function(str) { return str.toUpperCase() }, list);
// ['MAP', 'FILTER', 'FOLD']

var filter = function(pred, list) {
    return foldR(function(x, acc) { 
        return pred(x) ? cons(x, acc) : acc 
    }, [], list) 
}
filter(function(str) { return str.charAt(0) === 'M' }, list);
// ['Map']
